# Slice 2 Developer: Export Guardrails

## Metadata
- Owner: Origo Engineering
- Last updated: 2026-03-05
- Slice reference: S2 (`S2-G1`, `S2-G2`, `S2-G3`)

## Purpose and scope
- Documents guardrails that wrap the proven native export capability.
- Scope is export rights gating, immutable lifecycle audit events, queue backpressure, and failure classification.

## Inputs and outputs
- Rights input: `contracts/source-rights-matrix.json`.
- Legal signoff input: source-specific artifact path in the rights matrix.
- Audit output: append-only JSONL events at `ORIGO_EXPORT_AUDIT_LOG_PATH`.
- Failure-classification output: `error_code` and `error_message` in export status payload.

## Data definitions
- Rights states:
  - `Hosted Allowed`
  - `BYOK Required`
  - `Ingest Only`
- Audit event fields:
  - `sequence`
  - `occurred_at`
  - `event_type`
  - `export_id`
  - `previous_event_hash`
  - `payload`
  - `event_hash`
- Failure metadata file: `ORIGO_EXPORT_ROOT_DIR/<run_id>/error.json`.

## Source, provenance, freshness
- Rights decisions are resolved on each submit call from the rights matrix contract.
- Audit events are generated by API lifecycle points (submit accepted/rejected, status transitions).
- Failure classifications are persisted by Dagster export job failure path.

## Failure modes and error codes
- Rights gate fail-closed codes:
  - `EXPORT_RIGHTS_MISSING_STATE`
  - `EXPORT_RIGHTS_INGEST_ONLY`
  - `EXPORT_RIGHTS_BYOK_REQUIRED`
  - `EXPORT_RIGHTS_LEGAL_SIGNOFF_MISSING`
- Queue backpressure:
  - `EXPORT_QUEUE_LIMIT_REACHED`
- Audit failures:
  - `EXPORT_AUDIT_WRITE_ERROR`
- Terminal run failures:
  - persisted from `error.json`, fallback to `EXPORT_RUN_FAILED` or `EXPORT_RUN_CANCELED`.

## Determinism and replay notes
- Audit log is hash-chained and sequence-validated before every append.
- Any chain mutation (sequence/hash mismatch) fails loudly and blocks writes.
- Guardrail proof artifact: `spec/slices/slice-2-raw-export-native/guardrails-proof.json`.

## Environment and required config
- `ORIGO_EXPORT_MAX_CONCURRENCY`
- `ORIGO_EXPORT_MAX_QUEUE`
- `ORIGO_SOURCE_RIGHTS_MATRIX_PATH`
- `ORIGO_EXPORT_AUDIT_LOG_PATH`
- `ORIGO_EXPORT_ROOT_DIR`

## Minimal example
- Rights gate rejection:
  - rights state `Ingest Only` for requested dataset -> `409` with rights code.
- Queue backpressure rejection:
  - pending submissions >= `ORIGO_EXPORT_MAX_QUEUE` -> `503` with `EXPORT_QUEUE_LIMIT_REACHED`.
- Failed run classification:
  - `error.json` present -> status response includes persisted `error_code`/`error_message`.
