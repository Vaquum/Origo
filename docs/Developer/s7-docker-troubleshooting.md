# Slice 7 Developer: Docker Troubleshooting

## Metadata
- Owner: Origo Engineering
- Last updated: 2026-03-07
- Slice reference: S7 (`S7-P1`, `S7-P3`, `S7-P4`)
- Version reference: API `v0.1.2`, control-plane `v1.2.52`

## Purpose and scope
- Quick diagnostic runbook for common local Docker stack failures in Slice 7 proof flow.

## Inputs and outputs with contract shape
- Inputs:
  - service health state (`docker compose ps`)
  - recent service logs (`docker compose logs --tail 300`)
  - proof artifacts under `spec/slices/slice-7-docker-local-platform/`
- Outputs:
  - root-cause classification and exact failing step

## Data definitions (field names, types, units, timezone, nullability)
- `smoke-result.json`:
  - `status`: `passed | failed`
  - `failed_step`: nullable string step identifier
  - `exit_code`: integer process exit code

## Source/provenance and freshness semantics
- Failures are diagnosed from same-run artifacts generated by proof runner trap.
- No historical fallback sources are used in triage.

## Failure modes, warnings, and error codes
- `service_healthy` timeout:
  - inspect `docker-compose-tail.log` and per-service logs.
- Dagster export dispatch/status mismatch:
  - verify `ORIGO_DAGSTER_REPOSITORY_NAME` and `ORIGO_DAGSTER_LOCATION_NAME`.
- Query or export 404 for fixed window:
  - verify seed ingest succeeded and row counts in `seed-ingest-results.json`.

## Determinism/replay notes
- Replay mismatch is a hard failure and should block slice closeout.
- Root cause must be fixed; no tolerance mode is allowed for deterministic checks.

## Environment variables and required config
- Confirm all required vars from `.env.example` are present in local `.env`.
- In particular, verify:
  - `ORIGO_DOCKER_*` mapping vars
  - `ORIGO_DAGSTER_*` dispatch vars
  - `CLICKHOUSE_*` and guardrail vars used by API/control-plane

## Minimal examples
- Check health:
  - `docker compose ps`
- Inspect logs:
  - `docker compose logs --no-color --tail 300`
- Re-run full proof:
  - `scripts/s7_docker_stack.sh proof`
